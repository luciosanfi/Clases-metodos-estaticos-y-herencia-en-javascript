Continuacion de clases: propiedades estaticas , metodos estaticos y herencia.

Propiedades estaticas:

Estas Propiedades pueden ser utilizadas sin la necesidad de instanciar la clase, es decir, no hace falta crear un objeto de la clase, ya que estas propiedades pertenecen a la clase y no al objeto.
class Persona {
static bornDate = 1995;
de esta manera se declara una propiedad estatica y para acceder al valor de la misma simplemente:
console.log(Persona.bornDate);
Aqui demostramos que justamente no hace falta un objeto para acceder a la propiedad estatica ya que, se accede solamente con el nombre de la clase.

Set y get estaticos.

El setter estatico se utiliza de forma similar al comun solo que es para asignar el valor a una propiedad estatica y que en vez de utilizar this.propiedad tenemos que escribir el nombre de la clase.propiedad como:
Persona.bornDate
class Persona {
static bornDate = 1995;
static set date(bornDate){
persona1.bornDate = bornDate;
}
}
Y para acceder al valor de la misma podemos utilizar un getter tambien accediendo desde la clase y no desde un objeto
class Persona {
static bornDate = 1995;
static set date(bornDate){
Persona.bornDate = bornDate;
}
static get date(){
return Persona.bornDate;
}
}
const persona1 = new Persona();
console.log(Persona.bornDate);
Persona.date = 2000;
console.log(Persona.date);

Metodos estaticos:

Bien Los getters y setters no son otra cosa que metodos, asi que la aplicacion de los metodos estaticos en general, es la misma que la de los setters y getters simplemente escribimos la palabra static antes del nombre del metodo y cuando lo querramos utilizar, lo tenemos que hacer a traves de la clase y no del objeto.
class Persona {
static bornDate = 1995;
static set date(bornDate){
Persona.bornDate = bornDate;
static saludo(){
console.log("Hola te estoy saludando estaticamente");
}
Luego para llamar al metodo estatico:
Persona.saludo();

Herencia:

La herencia en javascript nos permite crear una clase nueva cuyos objetos tengan propiedades propias pero que tambien tengan, propiedades de una clase ya existente, la clase padre o superclase.
Por ejemplo vamos a heredar una clase de la clase persona. por logica las personas pueden ser taxistas.
class Taxista extends Persona{
numeroDeChapa = 0;
set nombres(nombre){
this.nombre = nombre;
}
get nombres(){
return "mi nombre es " + this.nombre;
}
set apellidos(apellido){
this.apellido = apellido;
}
get apellidos(){
return "mi apellido es " + this.apellido;
}
}
Entonces en el ejemplo vemos que tomamos Las propiedades nombre y apellido de la clase persona que ya existia y le agregamos la propiedad numero de chapa.
